Mermaid Syntax Rules
Apply these rules to every mermaid block. An invalid block is worse than a fallback table.
- Use only these diagram types: pie, xychart-beta, gantt, timeline, flowchart.
- The first line inside the block is the diagram type. The fence lines contain nothing but ```mermaid and ```.
- No Markdown formatting, HTML tags or %% comments inside mermaid blocks.
- Numeric values are plain numbers: no "~", "%", units, currency symbols or thousands separators. Units and estimate flags belong in the description bullets.
- pie: every slice label in double quotes; one "label" : value pair per line.
- xychart-beta: chart title and axis titles in double quotes; x-axis category labels in double quotes when they contain spaces; values are plain numbers.
- gantt: task names must not contain ":" or "," (replace with "-"); task IDs short and alphanumeric; dates strictly YYYY-MM-DD; durations like 60d; dependencies only as "after taskId"; status tags only done, active, crit, milestone.
- timeline: period and event text must not contain ":" (the delimiter).
- flowchart: short alphanumeric node IDs (A, B, N1); all display text in double quotes, as in A["Input"], C{"Valid?"}, subgraph SG1["Phase 1"] and -->|"Yes"|; never use lowercase "end" as a node ID.
- When content cannot satisfy these rules, use the documented fallback (HTML table or relationship lines) instead of guessing at syntax.
